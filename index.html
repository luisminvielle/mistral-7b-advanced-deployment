Taking Mistral-7B Further: Advanced Deployment for LLMs

By Luis Minvielle · October 2025 · View source on GitHub

I wanted to test how different GPUs work with Mistral-7B. So I integrated it with LangChain and then compared different pods to show how you can take deployment a step further with RunPod, a cloud computing platform designed specifically for AI.

In this guide

  1. How to Integrate LangChain with Mistral-7B for a Real-World Business Application
  2. How to Compare Inference Speed for Two Different GPU Types

If you've been reading the Learn AI with Me series, you know by now that you can easily deploy Mistral-7B on RunPod. It literally takes a single click. Now it's time to tweak it and optimize. This guide goes over advanced ways to change how inference workflows work right inside RunPod's infrastructure.

We'll skip the basic setup and focus on tuning for real-world use: getting responses with "streamed" text, comparing GPUs, understanding which metrics matter, and tinkering with features.

TGI and text-generation-webui: While this tutorial uses TGI, you could alternatively deploy via text-generation-webui on RunPod. Deploy with similar GPU specs and share the results.

How to Integrate LangChain with Mistral-7B for a Real-World Business Application

This step-by-step guide will show how I managed to integrate LangChain with this model. LangChain is the standard that we'll be using to elaborate the chain. The chain is what allows us to ask the query to the Mistral model. It's just four lines, but those are mighty four lines. For your reference, this is what LangChain looks like in my .py file:

# Create a prompt template
prompt_template = PromptTemplate(
  input_variables=["question"],
  template="""<s>[INST] {question} [/INST]""",
)

chain = prompt_template | llm | StrOutputParser()

Step 1: Choosing a Template

First, you must run the Mistral-7B on a pod. You can start with the Explore page and pick the Mistral 7B Instruct v0.1 – TGI API by Trelis. From there on, you should click on Deploy. From there on, you can choose your GPU.

I chose the RTX A4500, with a single GPU. Its specs are 12 vCPU and 62 GB RAM. I deployed one Mistral package (specifically Mistral 7B Instruct v0.1) from the Explore page. I had added a $10 credit and when the deployment finished, I was at $9.68, so the deployment cost me 32 cents. From a quick search online, I noticed that this GPU runs for around $2,000 each piece.

As per the logs, the download and deployment took me 50 minutes from start to finish (in which I didn't have to do nothing, just wait!). I suspect Docker might have cached the layers, so every identical deployment from this point on was considerably faster, in the order of just minutes.

Step 2: Adding the Hugging Face Token

What you should know is that Hugging Face now requires explicit permission to access the Mistral model. From what I tried, this is not optional. If you haven't done this, you'll get an error that indicates an unauthorized call.

To fix it, first, go to Hugging Face and accept Mistral's terms. Then, create an API token there. Inside your RunPod, make a new environment variable:

HUGGINGFACE_HUB_TOKEN

This step allows your pod to properly use the Mistral model. After this, you can connect to your pod with HTTP on port 8080, adding /generate to its web address, or connect with SSH for code.

Step 3: Configuring Your Endpoint

Configuring your endpoint is very easy on RunPod. On the pods' dashboard, you must choose your current pod and click on Connect. When you click on Connect, you'll be prompted with a message which says HTTP Services. If it doesn't show up at first, wait for it. That's the hyperlink to your endpoint.

Step 4: Integrating LangChain and Choosing Parameters

You now need to integrate LangChain to your Python file. The Python file is what you'll use to initialize the LLM and query it so it can answer your question.

The default endpoint here will be /generate. With /generate, you can ask the LLM to answer your query and send out a big chunk of text. There's a different way to do it and we'll get into it afterwards. This is what my endpoint looks like:

RUNPOD_ENDPOINT = "https://your-pod-id-8080.proxy.runpod.net/generate"

First, install every LangChain-related package:

pip install langchain

Or, if running inside a virtual environment, you can use Poetry. Your pyproject.toml file should include these lines:

"langchain (>=0.3.25,<0.4.0)",
"langchain-community (>=0.3.25,<0.4.0)",
"langchain-core (>=0.3.65,<0.4.0)",
"langchain-huggingface (>=0.3.0,<0.4.0)",

Import these functions in your code:

from langchain_huggingface import HuggingFaceEndpoint
from langchain_core.prompts import PromptTemplate
from langchain_core.output_parsers import StrOutputParser

You should also use this step to pick your parameters. I used moderate, standard parameters. One key part of my tests was that I always kept the parameters the same. That allowed me to compare inference times between GPUs:

# Initialize the LLM with your RunPod endpoint and inference parameters
llm = HuggingFaceEndpoint(
    endpoint_url=RUNPOD_ENDPOINT,
    max_new_tokens=512,
    top_k=10,
    top_p=0.95,
    typical_p=0.95,
    temperature=0.1,
    repetition_penalty=1.03,
)

Step 5: Running Your Python File

The integration is complete. If you have your Python file ready, you can now run it and LangChain and Mistral will answer your question. Here's the full program:

from langchain_huggingface import HuggingFaceEndpoint
from langchain_core.prompts import PromptTemplate
from langchain_core.output_parsers import StrOutputParser

RUNPOD_ENDPOINT = "https://your-pod-id-8080.proxy.runpod.net/generate"

# Initialize the LLM with your RunPod endpoint
llm = HuggingFaceEndpoint(
    endpoint_url=RUNPOD_ENDPOINT,
    max_new_tokens=512,
    top_k=10,
    top_p=0.95,
    typical_p=0.95,
    temperature=0.1,
    repetition_penalty=1.03,
)

# Create a prompt template
prompt_template = PromptTemplate(
    input_variables=["question"],
    template="""<s>[INST] {question} [/INST]""",
)

# Create a chain using the LangChain Expression Language (LCEL)
chain = prompt_template | llm | StrOutputParser()

# Ask a question
question = "what do you think of the hitchhikers guide to the galaxy"
print(f"Question: {question}")

# Get the response
response = chain.invoke({"question": question})
print("Response:")
print(response)

The first answer, the one about serverless and how it can let a business scale, took around 9 seconds to roll in. The second one, about life, the universe, and everything, took 6 seconds. This is also how you can see how fluid LLMs are.

Step 6: Adapting the Answers so that they Stream

The initial response technique was synchronous. This means that you get a chunk of text. There's a specific endpoint for streaming: /generate_stream. This is the updated asynchronous code:

# Get the response in chunks, as it streams asynchronously
print("Response (streaming):")

full_response = ""
async for chunk in chain.astream({"question": question}):
    print(chunk, end="", flush=True)
    full_response += chunk

I ended up getting the same answers but "streamed." For your reference, when I started streaming the answers, I was spending $0.25/hour on my RunPod environment. The response times were the same for this GPU if the text was streamed or not, but getting a streamed endpoint means that the answer starts showing up much earlier.

If you're about to integrate this LangChain + Mistral setup to a customer support chatbot, go for the /generate_stream endpoint. That's an easy entry into a superior LLM performance.

How to Compare Inference Speed for Two Different GPU Types

RunPod currently offers around 25 different GPUs you can choose from. This means that you can draw around 300 comparisons between GPUs. After reading this tutorial, you'll be able to compare any of these GPUs for inference time. You'll also be able to spot easily which GPUs perform better for your business application.

For this comparison, I used two GPU types with completely distinctive setups so that the differences would jump out:

  1. 1 x RTX A4500 — the same pod I used for the previous section. This GPU runs for $0.25/hour.
  2. 1 x L40S — 16 vCPU and 94 GB RAM, bulkier than the other pod. This GPU runs for $0.86/hour.

Keep in mind, I never ran out of my initial $10, even if I ran extensive screenings with plenty of other pods that didn't make it into this article.

Step 1: Deploying Your Pods

To deploy your new pods, follow the same steps I outlined for the previous tutorial. You don't need to use the same GPUs I chose here. But use the same setup so you can draw your own comparisons: Mistral-7B on both.

Step 3: Understanding which Measures Matter

There are three performance metrics you'll be getting on your terminal:

  1. Time to First Token (TTFT) — The time it takes for an LLM to generate and output the very first piece of its response after receiving a prompt.
  2. Total Generation Time — The entire duration from when an LLM receives a prompt until it has completed and outputted its full, final response.
  3. Characters per Second — The rate at which an LLM "writes" characters in its output.

The measure that really matters is total generation time. Because if there's a human on the other end of the LLM, the UX will be contingent on this measure more than on the others.

Step 4: Running Your Code and Checking the Results

When I asked these two GPUs about the theory of relativity, they gave me similar answers with very different metrics:

Metric 1 x L40S 1 x RTX A4500
Time to First Token (TTFT) 2.1403 s 1.4494 s
Total Generation Time 8.3748 s 11.3774 s
Characters per Second 204.03 145.59

I then went on to ask more questions. The difference when asking about serverless was also noticeable.

When I asked about life, the universe and everything, though, the total generation time was incredibly similar. This shows how non-deterministic a LLM architecture can be. Three different questions, same setup, relatively similar answer length, and incredibly different results.
